| :--- | :--- |

| **面向岗位** | 前端开发、游戏策划、测试、UI/UX设计 |
| **技术栈建议** | HTML5 Canvas, TypeScript, Webpack |

---

## 1. 产品概述

### 1.1 背景
随着Web游戏市场的竞争加剧，传统贪吃蛇的单一玩法已难以满足用户对深度和个性化的需求。V2版本旨在通过引入**参数化配置**、**动态生态（野怪）**以及**高自由度的外观系统**，将游戏从单纯的“闯关”升级为具有Roguelike元素和收集乐趣的网页游戏。

### 1.2 核心目标
- 引入**移动反应时间**配置，增加操作手感的微调维度。
- 增加**随机生成的敌对/中立蛇**，丰富关卡动态变化。
- 修改核心规则：**取消自杀判定**，允许蛇身互相缠绕，降低挫败感。
- 设计**10种以上**独特的蛇身装饰样式，增强视觉表现与用户粘性。

---

## 2. 需求详述

### 2.1 核心机制变更 (Breaking Changes)
- **取消自杀碰撞**：
    - **规则**：蛇头穿过自己的身体不再导致死亡或游戏结束。
    - **目的**：降低新手门槛，允许玩家采用“绕圈”等高阶策略，增加容错率。
- **动态实体引入**：
    - 关卡中除了玩家外，会随机生成NPC蛇（Wild Snakes）。
    - NPC蛇会竞争食物，增加关卡的不确定性和重玩价值。

### 2.2 新增配置项：移动反应时间 (Input Latency Config)
为了满足不同玩家的操作习惯和设备性能，新增游戏配置参数。

| 配置项 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| **moveReactionTime** | Number (ms) | 100 | **移动反应时间**。指按下方向键后，蛇头实际改变方向所需的延迟时间。 |
| **应用场景** | - | - | - **休闲模式**：200ms-300ms，容错率高。<br>- **硬核模式**：50ms-80ms，极速响应。<br>- **第8关（镜像）**：强制锁定为150ms，增加混乱感。 |

### 2.3 随机实体：野生蛇 (Wild Snakes)
- **生成逻辑**：每局开始后第10秒，或在食物少于3颗时，随机在空旷区域生成1条野生蛇。
- **数量上限**：同屏最多存在 2 条野生蛇。
- **行为模式**：
    - **觅食型 (Green)**：追逐最近的豆子，无视玩家。
    - **游荡型 (Gray)**：随机游走，阻碍玩家路径。
    - **攻击型 (Red)**：在第10关出现，主动拦截玩家（V2新增）。
- **交互规则**：
    - 玩家头部撞到野生蛇身体 -> **玩家无事，野生蛇死亡并掉落金豆**。
    - 野生蛇头部撞到玩家身体 -> **野生蛇死亡**。
    - 野生蛇之间互不影响。

### 3.2 UI展示
- **皮肤选择界面**：在主菜单或暂停菜单增加“换装”入口。
- **解锁机制**：通过累计过关星数或达成特定成就（如“不碰墙通关”）解锁皮肤。

## 5. 技术实现要点 (V2)

### 5.1 输入缓冲系统 (Input Buffer)
由于引入了 `moveReactionTime`，不能直接响应按键。
- 建立一个按键队列。
- 当按下方向键时，记录时间戳。
- 在游戏循环中，检查当前时间与按键时间的差值是否大于 `moveReactionTime`，若是则执行转向。

### 5.2 碰撞层级重构
- **Layer 0**: 边界 & 墙壁 (Block All)
- **Layer 1**: 玩家蛇头 (Collide with Layer 0, Layer 3, Layer 4)
- **Layer 2**: 玩家蛇身 (**不再检测与Layer 1的碰撞**) 
- **Layer 3**: 野生蛇头 (Collide with Layer 0, Layer 1, Layer 3)
- **Layer 4**: 野生蛇身 (Collide with Layer 1)

### 5.3 NPC 蛇 AI 逻辑
- **寻路算法**：使用简化的 A* 算法或贪婪最佳优先搜索（Greedy BFS）寻找最近的食物。
- **避障**：当检测到前方有墙壁或玩家蛇头时，随机选择左转或右转规避。

---

## 6. 验收标准 (V2 Update)

1. **反应时间**：修改配置文件中 `moveReactionTime` 的值，蛇的转向延迟应有明显体感差异。
2. **穿身机制**：玩家蛇头穿过自己身体3次以上，游戏不应触发Game Over。
3. **野生蛇**：第2关开始后，地图上必须出现非玩家控制的蛇，且能正常吃豆或移动。
4. **皮肤系统**：切换皮肤后，蛇的材质、颜色或特效应立即更新，且在关卡重启后保留。
5. **性能**：同屏存在2条野生蛇+玩家蛇+大量粒子特效时，帧率仍保持在55+FPS。
